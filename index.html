CentOS 7 安装非最新版MySQL

版本版本：CentOS Linux release 7.9.2009 (Core)
内核版本：3.10.0-1160.24.1.el7.x86_64
截至写稿前，MySQL已经更新到MySQL8.0版本，但是在实际生产环境中，有工作者不需要安装最新版，而是安装一些之前版本的稳定版，例如MySQL5.7.
本文就以在CentOS 7.9系统上安装MySQL 5.7。
若要安装非最新版本的mysql，则需要下载对应的rpm包，使用此方式进行安装。

一、MySQL Yum存储库

1.1 下载MySQL Yum存储库

MySQL Yum存储库提供了一种简单方便的方法，可以使用Yum使用最新的软件包来安装和更新MySQL产品。

默认情况下，下面提供的存储库软件包将提供MySQL Server 8.0。通过编辑软件包安装的回购设置文件可以获得其他版本。

  1. 转到https://dev.mysql.com/downloads/repo/yum/上的MySQL Yum存储库的下载页面，下载对应版本的存储库，然后上传至Linux系统中 。
  2. 选择并下载适用于您平台的发行包，下载对应版本的存储库，然后上传至Linux系统中 。。
  3. 也可以直接在Linux终端下执行：wget https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm进行下载

1.2 添加存储库

  1. 使用以下命令安装下载的发行包：
shell> rpm -Uvh mysql80-community-release-el7-3.noarch.rpm

一旦在系统上安装了发行包，通过yum update 命令进行的所有系统范围内的更新（或对于启用dnf的系统进行dnf升级）都将自动升级系统上的MySQL软件包，并且还会替换任何本机的第三方软件包（如果有的话）在MySQL Yum存储库中找到它们的替代品。

查找要配置的子存储库的条目，然后编辑enabled选项。指定 enabled=0禁用子存储库或 enabled=1启用子存储库。例如，要安装MySQL 5.7，请确保您具有enabled=0上面针对MySQL 8.0的子存储库条目，并且具有 enabled=1针对5.7系列的条目：

您应该随时只为一个发行系列启用子存储库。如果启用了多个发行系列的子存储库，那么Yum将使用最新的系列。

shell> yum repolist enabled | grep mysql
vim /etc/yum.repos.d/mysql-community.repo
# Enable to use MySQL 5.7
[mysql57-community]
name=MySQL 5.7 Community Server
baseurl=http://repo.mysql.com/yum/mysql-5.7-community/el/7/$basearch/
enabled=1	# 我需要安装5.7版本，所以这里改成1
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-mysql

[mysql80-community]
name=MySQL 8.0 Community Server
baseurl=http://repo.mysql.com/yum/mysql-8.0-community/el/7/$basearch/
enabled=0	# 禁用掉8.0版本
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-mysql

二、安装MySQL

2.1 安装MySQL

shell> yum install mysql-community-server	# 将安装MySQL服务器的软件包以及其他必需的软件包。

2.2 启动MySQL

shell> systemctl start mysqld
shell> systemctl status mysqld
shell> systemctl enable mysqld

三、MySQL服务器初始化

自MySQL 5.7起：在服务器初次启动时，假定服务器的数据目录为空，则会发生以下情况：

  • 服务器已初始化。
  • SSL证书和密钥文件在数据目录中生成。
  • 该validate_password插件安装并启用。
  • 'root'@'localhost' 创建 一个超级用户帐户。设置超级用户的密码并将其存储在错误日志文件中。要显示它，请使用以下命令：

3.1 获取root初始密码

shell> grep 'temporary password' /var/log/mysqld.log

3.2 修改root密码

通过使用生成的临时密码登录并尽快更改超级用户帐户的root密码，以更改root密码：

shell> mysql -uroot -p
mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY 'MyNewPass4!';	# 修改root密码

MySQL的 validate_password 插件是默认安装的。这将要求密码至少包含一个大写字母，一个小写字母，一位数字和一个特殊字符，并且密码总长度至少为8个字符。
